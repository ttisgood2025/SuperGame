# 《萌宠三叠消消乐》游戏设计文档（Cocos Creator 3.8.2）

## 1. 游戏定位
- **类型**：过关型三叠消除（Tile Match 3）
- **平台**：微信小游戏/抖音小游戏（可扩展 H5）
- **受众**：休闲用户、女性用户、轻度策略用户
- **单局时长**：1-3 分钟

## 2. 核心玩法
### 2.1 规则
1. 场上有多层叠放的宠物牌。
2. 玩家仅可点击“未被遮挡”的牌（上方无遮挡）。
3. 点击后牌进入底部槽位（默认容量 7）。
4. 槽位中同类型牌达到 3 张后自动消除。
5. 槽位被填满且无法消除时失败。
6. 清空场上所有牌则过关。

### 2.2 失败与复活
- 默认失败条件：槽位满 7 张。
- 复活机制：观看激励广告后 +2 槽位且自动整理一次。
- 复活限制：每局最多 1 次（前期可给 2 次用于新手留存）。

## 3. 关卡结构（60 关）
### 3.1 章节划分
- 第 1 章（1-15）：新手引导，牌种少，层数低。
- 第 2 章（16-30）：加入锁定牌、冰冻牌等轻障碍。
- 第 3 章（31-45）：复杂堆叠结构与限步目标。
- 第 4 章（46-60）：混合障碍、动态目标、较高容错压力。

### 3.2 难度参数维度
- 可用宠物种类数（6→18）
- 总牌数（36→144）
- 堆叠层数（2→6）
- 障碍占比（0%→20%）
- 道具掉落概率（高→低）

## 4. 上瘾机制设计（核心循环）
### 4.1 即时反馈
- 三消触发：弹跳、粒子、音效、连消倍率提示。
- 胜利瞬间：星级结算 + 宝箱进度增长。
- 失败提示：给出“差一点”文案和复活入口。

### 4.2 中期目标
- 连胜奖励：3 连胜、5 连胜给额外金币/道具。
- 每日任务：通关 3 关、触发 5 次连消、使用 1 次道具。
- 星星收集：用于解锁“萌宠家园”装饰内容。

### 4.3 长线成长
- 萌宠图鉴：解锁不同宠物皮肤。
- 家园装修：关卡产出“饼干”货币，购买家具。
- 周赛排行：按 7 日最高章节进度排名。

## 5. 经济系统
- **软通货**：金币（关卡奖励 + 任务奖励）
- **硬通货**：钻石（活动/付费）
- **消耗点**：
  - 道具：撤回、洗牌、临时扩槽位
  - 复活：广告优先，钻石兜底

## 6. 商业化建议
- 插屏：每 2-3 关一次（避免首日过密）
- 激励广告：复活、双倍奖励、免费道具
- 小额内购：新手礼包（去广告 + 钻石 + 道具）

## 7. Cocos Creator 3.8.2 技术实现建议
### 7.1 模块划分
- `GameController`：状态机（准备/进行/胜利/失败）
- `BoardManager`：牌堆生成、遮挡关系、可点击判断
- `SlotManager`：底部槽位、三消结算、满槽判定
- `LevelManager`：关卡配置加载与进度推进
- `EconomyManager`：金币、道具、广告奖励结算

### 7.2 数据驱动
- 使用 JSON 配置关卡参数（见 `config/levels.json`）。
- 用 Scriptable-like 配置（TS 接口 + 本地 JSON）减少硬编码。

### 7.3 性能优化
- 卡牌对象池（NodePool）复用。
- UI 合批：同图集/同材质。
- 特效降级：低端机减少粒子数量。

## 8. 美术与体验规范（萌宠 2D）
- 主色建议：奶油白、浅橙、薄荷绿。
- 形状建议：圆角卡牌 + 柔和投影。
- 动效节奏：点击反馈 < 80ms，消除演出 < 300ms。
- 音效策略：轻快短促，不连续轰炸。

## 9. 版本路线图
- **v0.1（2 周）**：核心玩法 + 20 关 + 基础 UI。
- **v0.2（2 周）**：扩展至 60 关 + 障碍系统 + 道具。
- **v0.3（2 周）**：经济系统 + 活动任务 + 广告接入。
- **v1.0（上线）**：家园系统 + 数据埋点 + AB 测试。
